## 1. 系統定位與邊界

### 1.1 產品定位

本系統是一個 **多空強度預測指標（Market Intelligence Indicator）**，預測 BTC 永續合約未來 4 小時的方向與幅度，以圖表形式可視化並透過 Telegram 推送。

**明確不是**：
- 交易策略或自動下單系統
- 進出場規則 / TP / SL / 倉位管理 / 槓桿控制
- 交易績效最大化或回測框架

**評估以預測品質為準**：IC、ICIR、方向準確率、calibration monotonicity、Strong 訊號勝率。

### 1.2 核心輸出（每小時整點產出一次）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `pred_return_4h` | float | 迴歸模型直出的 signed 4h path return 預測 |
| `pred_direction` | enum | `UP` / `DOWN` / `NEUTRAL`（由 rolling percentile tier 決定） |
| `strength_score` | enum | `Strong` / `Moderate` / `Weak` |
| `confidence_score` | 0–100 | 複合信心分數（見 §8.1） |
| `mag_pred` | float | |return_4h| 預測幅度（σ 還原後的 return scale） |
| `dir_pred_ret` | float | Direction 迴歸模型原始輸出（signed path return） |
| `dir_prob_up` | 0–1 | `sigmoid(dir_pred_ret × 200)` 合成值（legacy 相容，非校準機率） |
| `bull_bear_power` | -1 至 +1 | 跨源情緒綜合指標 |
| `regime` | string | `CHOPPY` / `TRENDING_BULL` / `TRENDING_BEAR` / `WARMUP` |

### 1.3 核心 target

```python
y_return_4h = close.shift(-4) / close - 1
```

所有特徵計算均為 **trailing-only**（無前視偏差）。`build_live_features()` 同時供訓練與生產使用，保證 train/serve 一致性。

### 1.4 系統規模現狀

| 項目 | 數值 |
|------|------|
| 資料量 | ~4350 bars 1h K 線（~181 天） |
| 總特徵數 | 130+ 工程特徵（12 群組） |
| Direction 生產特徵 | 136（XGBRegressor） |
| Magnitude 生產特徵 | 87（XGBRegressor） |
| 模型數 | 2（Direction-Reg + Magnitude，各 1 個 XGBoost） |
| 資料源 | Binance REST + Coinglass v4（24 端點）+ Deribit（2 端點） |
| 預測視窗 | 4 小時 |
| 更新頻率 | 每小時整點（:02 cron 觸發） |

## 3. 資料來源層

### 3.1 Binance 資料

**三個 endpoints**（`indicator/data_fetcher.py`）：

| Endpoint | 用途 | 頻率 |
|----------|------|------|
| `/fapi/v1/klines` | 1h OHLCV + taker buy volume | 每小時 500 bars |
| `/fapi/v1/depth` | Order book L20 深度 | 即時快照（每小時） |
| `/fapi/v1/aggTrades` | 聚合成交（近 ~1000 筆） | 即時抓取 |

**Service 2 WebSocket**：持久連線接收 tick 級 aggTrade，正規化後寫入 `flow_bars_1m`。

### 3.2 Coinglass API v4

**24 個 endpoints**，分兩類：

**時序類（15 個）**— 原生 1h：OI、清算、Long/Short、Funding、Taker、Coinbase Premium、Bitfinex Margin、Spot CVD、OI Coin Margin 等。

**快照類（9 個）**— 即時值（只寫最後一根 bar）：whale_index、fear_greed、puell_multiple、ahr999 等。

**關鍵設計**：使用 `merge_asof(direction='backward')` 精確對齊到 1h bar。

### 3.3 Deribit Public API

**2 個 endpoints**：
- `public/get_volatility_index_data`：DVOL 波動率指數（hourly OHLC）
- `public/get_book_summary_by_currency`：選擇權 summary（IV、OI per strike）

### 3.4 Backfill Pipeline

`research/backfill_all_parquet.py` — 4 步驟統一 backfill：
1. Binance klines（增量 pagination）
2. CG 既有 7 端點
3. CG 新增 7 端點
4. **Deribit DVOL**（2026-04-16 加入，增量 pagination + merge_parquet 去重）

`ensure_fresh(max_age_hours=6)` 掃描所有 parquet，任何一個超齡就觸發全量 backfill。

### 3.5 Timestamp 格式陷阱

不同 endpoint 可能用不同 unit。解析時必須自動偵測：
```python
unit = "s" if sample_ts < 1e12 else "ms"
```

## 5. 特徵工程管線

### 5.1 特徵總覽（12 群組，130+ 特徵）

所有特徵計算為 **trailing-only rolling**，嚴格禁止 look-ahead。`build_live_features()` 同時供訓練和生產使用。

| 群組 | 特徵數 | 代表性特徵 |
|------|--------|-----------|
| A. 價格與波動率 | ~15 | `log_return`, `realized_vol_20b`, `return_skew`, `vol_kurtosis`, `return_lag_1..10` |
| B. Taker 流 | ~12 | `taker_delta_ratio`, CVD 衍生, `taker_delta_ma_{4h,24h}` |
| C. CG OI | ~15 | `cg_oi_close_pctchg_{4h,8h,12h,24h}`, `cg_oi_range_zscore` |
| D. CG 清算 | ~8 | `cg_liq_imbalance`, `cg_liq_surge`, `cg_liq_cascade` |
| E. CG Long/Short | ~10 | `cg_ls_ratio`, `cg_gls_ratio`, `cg_ls_divergence` |
| F. CG Funding | ~8 | `cg_funding_close_zscore`, `oi_weighted_funding` |
| G. CG Taker | ~6 | `cg_taker_delta`, `cg_taker_ratio` |
| H. CG 額外 | ~15 | `cg_coinbase_premium`, `cg_bfx_margin_ratio`, `cg_spot_cvd` |
| I. Order Flow Toxicity | ~6 | VPIN 衍生、adverse selection proxy（IC=+0.071） |
| J. 自訂 Alpha | ~8 | `impact_asymmetry`（IC=-0.071）, `post_absorb_breakout`（mag IC=0.191） |
| K. Deribit 選擇權 | ~6 | `bvol_close`, `bvol_zscore`, `atm_iv`, `skew_25d` |
| L. 時間編碼 | 4 | `hour_sin`, `hour_cos`, `weekday_sin`, `weekday_cos` |

### 5.2 Interaction 特徵設計規則

**禁止 `feature × sparse_indicator`**：當 indicator 非零比例 < 30% 時，乘出來的特徵被 sparsity pattern 主導。

**正確寫法**：
- `feature × (1 - is_dead_regime)` — 屏蔽無效 regime
- 把 indicator 直接當 feature，讓 XGB 用 tree split 自己決定 conditional rule

## 6. Dual Model ML 管線

### 6.1 架構概覽（v7.1 Direction-Reg）

```
┌─────────────────────────────┐      ┌─────────────────────────────┐
│ Direction-Reg Model         │      │ Magnitude Model             │
│                             │      │                             │
│ XGBRegressor                │      │ XGBRegressor                │
│ target: signed path return  │      │ target: y_vol_adj_abs       │
│         (4h TWAP)           │      │   = |ret_4h| / realized_vol │
│ features: 136               │      │ features: 87                │
│ output: dir_pred_ret        │      │ output: mag_pred_sigma      │
│   (continuous signed value) │      │   (σ units, × rvol → return)│
│                             │      │                             │
│ Tier decoding:              │      │                             │
│   rolling 500-bar quantile  │      │                             │
│   top 5% → Strong           │      │                             │
│   top 15% → Moderate        │      │                             │
└──────────────┬──────────────┘      └──────────────┬──────────────┘
               │                                    │
               └────────────┬───────────────────────┘
                            ▼
                    複合訊號生成（§8）
```

### 6.2 Direction-Reg（v7.1，2026-04-15 上線）

**Target**：`y_path_ret_4h`（signed 4h TWAP return，連續值）

**與 v7.0 Binary Classifier 的差異**：

| | v7.0 Binary | v7.1 Regression |
|---|---|---|
| 模型類型 | XGBClassifier | XGBRegressor |
| 輸出 | P(UP) ∈ [0,1] | signed return ∈ R |
| Tier 判定 | 固定門檻 (0.60/0.40) | rolling percentile (top 5%/15%) |
| 死區 | σ-based label NaN | 所有樣本都訓練，tier 由分位決定 |
| 特徵數 | 29（pruned） | 136（full set） |
| 校準 | 原生 P(UP) | `sigmoid(pred_ret × 200)` 合成（非校準） |

**Rolling Percentile Tier Decoding**：
```python
# 500-bar expanding buffer
# Strong: |pred| >= top 2.5% quantile (symmetric two-tailed → top 5%)
# Moderate: |pred| >= top 7.5% quantile (symmetric → top 15%)
# Cold start (<100 bars): fallback thresholds from direction_reg_config.json
```

**XGBoost 參數**：
```python
REG_PARAMS = {
    "objective": "reg:squarederror",
    "max_depth": 4,
    "learning_rate": 0.05,
    "n_estimators": 300,
    "subsample": 0.8,
    "colsample_bytree": 0.7,
    "min_child_weight": 10,
}
```

### 6.3 Magnitude Regressor

**Target**：`y_vol_adj_abs = |return_4h| / realized_vol`（σ 正規化，推論時乘回 `realized_vol_20b` 還原）

**特色 alpha 特徵**：
- `post_absorb_breakout`：mag IC = 0.191（magnitude 第一重要）
- `impact_asymmetry`：IC = -0.071

**OOS 績效（77-fold walk-forward）**：
- Spearman IC: 0.35, ICIR: 1.54
- Top/Bottom decile ratio: 3.30
- Monotonicity: 1.0（完美單調）

### 6.4 Walk-Forward 交叉驗證

```python
walk_forward_splits(
    n=len(df),
    initial_train=288,   # 12 天暖機
    test_size=48,         # 每折測 48h = 2 天
    step=48,              # 每 2 天滑動
    purge=4, embargo=4,   # 防止 horizon leakage
)
# 結果：77 折，擴展窗口（始終從 bar 0 開始）
```

## 7. 即時推論引擎

### 7.1 預測流程（每小時 :02 cron 觸發）

```
indicator/app.py::hourly_cycle()
│
├─ 1. fetch_binance_klines(limit=500)
├─ 2. fetch_coinglass_all(24 endpoints)
├─ 3. fetch_deribit_dvol() + fetch_deribit_options()
│
├─ 4. build_live_features(klines, cg, deribit) → 130+ 特徵
│
├─ 5. IndicatorEngine.predict(features)
│     ├─ 特徵對齊（direction 136 + magnitude 87 各自取子集）
│     ├─ Regime 偵測（trailing-only expanding percentile）
│     ├─ dir_pred_ret = direction_xgb.predict(X_dir)       ← 迴歸
│     ├─ dir_prob_up = sigmoid(dir_pred_ret × 200)          ← 合成
│     ├─ mag_pred_sigma = magnitude_xgb.predict(X_mag)
│     ├─ mag_pred = mag_pred_sigma × realized_vol_20b       ← σ 還原
│     ├─ Tier: rolling percentile → Strong/Moderate/Weak
│     ├─ confidence_score（見 §8.1）
│     └─ Bull/Bear Power（5 個 CG z-score 截斷平均）
│
├─ 6. Re-predict all historical bars with current model（full rescore）
├─ 7. 寫入 indicator_history（parquet + MySQL）
│
├─ 8. Strong → SHAP driver 分析 + signal_tracker 記錄
│
├─ 9. render_chart() → 4-panel PNG
├─ 10. Telegram push (PNG + caption)
└─ 11. Watchdog 健康檢查
```

### 7.2 IndicatorEngine 類別

`indicator/inference.py`：

- **Mode 偵測**：`direction_reg_config.json` 存在 → regression mode，否則 binary classifier
- **SHAP 相容**：`signal_explainer.py` 偵測 regression mode，用 `XGBRegressor` 載入 + SHAP sign flip for DOWN

### 7.3 `dir_prob_up` Legacy 相容

Direction-Reg 的 `dir_prob_up` 是 `sigmoid(dir_pred_ret × 200)`，**不是校準機率**。
- 保持 monotonicity（pred_ret 越大 → dir_prob_up 越接近 1）
- 但不能拿來做：binary entropy、calibration diagram、probability display
- `entropy_tools.py` 已從 runtime 移除（2026-04-16）
- Telegram caption 已改顯示 `Mag: 0.032%` 取代 `P(UP): 85%`

## 8. 訊號生成與 Gating

### 8.1 Confidence 分數（v7.1）

```python
# Direction-Reg mode:
ref = max(|up_strong_cutoff|, |dn_strong_cutoff|)
ret_score = min(|dir_pred_ret| / ref, 1.0) ** 0.6 × 80    # 80 分來自方向
mag_bonus = (mag_percentile / 100) × 20                     # 20 分來自幅度
confidence_score = clip(ret_score + mag_bonus, 0, 100)
```

**強度分級**：
- `Strong`：confidence >= 80（約佔 5% bars）
- `Moderate`：confidence >= 65（約佔 15% bars）
- `Weak`：confidence < 65（約佔 85% bars）

### 8.2 方向判定

Direction-Reg 模式下，方向不再由固定門檻決定，而是由 **rolling 500-bar percentile tier** 決定。只有達到 Moderate 以上 tier 的 bar 才被判定為 UP/DOWN，其餘為 NEUTRAL。

### 8.3 Strong 訊號觸發

當 `strength_score == "Strong"` 時額外觸發：
1. **SHAP driver 分析**：TreeExplainer（支援 XGBRegressor）
2. **Telegram 文字告警**：含 top 5 驅動因子
3. **signal_tracker 記錄**：4h 後自動回填 outcome

## 16. 優化路線圖

### 16.1 近期可執行（高 ROI，1-2 週內）

#### A. Magnitude 模型定期重訓
- **問題**：Mag walk-forward OOS IC 從 Nov 0.31 腰斬到 Mar 0.10（concept drift）
- **方案**：每 2-3 週用最新 features_all 重訓 Mag，保持 in-distribution
- **預期效果**：IC 回升到 0.25+，confidence 分數更準

#### B. Direction-Reg 特徵選擇
- **現況**：136 特徵全塞（sample:feature ratio ~30:1，偏低）
- **方案**：用 `auto_pruner.py` 做 IC-filter + VIF redundancy removal，壓到 50-70 個
- **預期效果**：減少 overfitting，AUC 可能從 0.57 升到 0.59-0.60

#### C. Calibration 重新量測
- **現況**：切模型後還沒累積足夠 clean samples
- **條件**：等新模型跑到 n>=300 samples（~12 天），再跑 `calibration_check.py --since 2026-04-15`
- **後續**：如 ECE > 0.05，用 isotonic regression 校準

### 16.2 中期探索（需實驗驗證）

#### D. Horizon 延伸至 8h
- **動機**：4h 方向 AUC 天花板 ~0.57；更長 horizon 的噪音更少
- **方法**：修改 target 為 `close.shift(-8) / close - 1`，walk-forward 驗證
- **風險**：更長 horizon 的 signal 可能更稀疏，三角形間隔拉大

#### E. Vol-Adjusted Direction Target
- **動機**：低 vol 期的 ±0.1% 回報和高 vol 期的 ±0.1% 意義不同
- **方法**：`y_dir = return_4h / realized_vol`，讓模型在 σ-scale 上決定方向
- **預期**：過濾低 vol whipsaw，AUC 在有意義子集上提升

#### F. Mag 作為 Direction Gate
- **現況**：Mag 只影響 confidence 分數的 20% 權重
- **數據**：Mag >= p80 時 Direction Strong WR 從 72.9% → 77.6%（+4.7pp）
- **方案**：加 gate — 只在 Mag >= p80 時才發 Strong 訊號
- **權衡**：WR 提升 +5pp，但 signal 數量減半（~14 次/月 → ~7 次/月）

### 16.3 長期方向（需架構變更）

#### G. 多 Horizon 集成
- 同時訓練 4h / 8h / 12h 三個 Direction model
- 三個模型一致時才發 Strong，不一致時降級或 NEUTRAL
- 增加資訊量但增加維護複雜度

#### H. Online Learning / Incremental Update
- 每次新 bar 到達時對模型做小幅 fine-tune（XGBoost 支援 `process_type='update'`）
- 解決 concept drift 而不需完整 retrain
- 風險：catastrophic forgetting、需要 validation guard

#### I. 新資料源
- **鏈上數據**：exchange netflow、whale alert、stablecoin flow
- **社群情緒**：Twitter/X 情緒分析、Reddit sentiment
- **宏觀**：DXY、US10Y、SPX correlation features
- 每個新源都必須先跑 IC 驗證

## 17. 已知問題與關鍵設計決策

### 17.1 核心原則

1. **無前視偏差**：trailing-only rolling
2. **歷史與即時一致性**：`build_live_features()` train/serve 共用
3. **時間對齊精準**：`merge_asof backward`
4. **預測導向評估**：IC、方向準確率、calibration
5. **新特徵先回測**：IC/AUC 驗證才加入
6. **圖表同步規則**：靜態與互動必須同步
7. **評估前 sanity check**：確認同一模型版本 + 乾淨數據

### 17.2 已知問題
